Rescue-Trac Hi-Fi Prototype

Rescue-Trac is a concept for a pet-finder app. This was done as a group project where I worked as the team lead and the main UI/UX designer. It was created for a User-Centered Design class, and the purpose of the assignment was to take an idea for a mobile app from a basic flowchart of ideas to a hi-fi interactive prototype. Throughout the course of this semester-long project, I learned many important UX protocols and methods, as well as ways to gather and implement user feedback.

The app's main purposes are to help users find pets to adopt and to help other users find lost pets. Key features are:

  • Notifications
  • The ability to follow rescue shelters and foster homes that the user is interested in
  • A button on pet profiles to message the shelter directly in the app
  • A button on lost/found pet posts to message the owner/finder of a pet directly in the app
  • Optional inclusion of a telephone link for pet profiles or posts
  • Bookmarking functionality to save a list of pet profiles a user is interested in

The color scheme is playful and friendly while maintaining a calm atmosphere. The teals and greens provide a feeling of tranquility while the orange accents brighten up the palette and provide a contrast to the cooler hues.

Icons such as the filter and the user represent expected functionality for better user experience. Touch targets are kept large enough to prevent misclicks and color contrast is demonstrated for good accessibility.

Social Platform Concept

Concept: This endeavor helped me learn more about how social media sites are put together. The mobile app would be called "Audiophile", which would take inspiration from other platforms such as Reddit and Twitter. It would be a place for music-lovers to discuss different genres, share playlists, recommend new artists, and discuss new releases. Some of the app's features would include live-streaming, event-organizing, and Sub-Hubs, which are similar to Reddit's concept of subreddits.

Imagery: logo drawn in Ibis Paint X. An audiophile is someone who is enthusiastic about recording and listening to high-fidelity music. The logo I designed is a stylized version of a vinyl record, which is not just a common symbol of music but also a popular listening medium among audiophiles, making it a good pick for the app's logo. The color scheme includes red because of it's symbolism of passion and excitement, which are common feelings people have when listening to and talking about their favorite music.

Interface: Designed in Picsart. The top app bar includes a profile button, the app logo, and a side drawer. The bottom tab bar includes buttons for the home feed, search/discovery page, post creation, livestreaming, and messages. The design is high contrast, which is beneficial for the app's style goals and for accessibility.

To-Do App Mockup

This app design utilizes a lively interface for completing an often boring activity. To keep better track of their tasks, users can make an account for use across devices. Demostrated features in mockup:

  • User icon reveals Log Out button
  • "Filter..." button reveals three filtering options
  • Meatballs menu button reveals Edit and Delete options

Light green is chosen as the background color to help the user feel calm and reassured in their task creation. Teal goes nicely as an accent color and a gradient is used to give the design a more interesting look.

The layout resembles a traditional task list that would be put on paper while also including features for ease of use, such as a calendar to set due dates and the ability to filter and edit previously set tasks.

Car Inventory Management System

In this app, users can input car specs into a form and see them output into a list at the bottom.

Key Features
  • Price sorting in ascending order
  • Price averaging
  • Price totaling
  • Image zoom
  • Clearly labeled buttons
Skills Highlighted
  • Outputting images to page through image address
  • Sorting entries
  • Various math calculations
  • Zoom functionality on hover

While I was creating this project, I had trouble with sorting the list of entries. I was trying to replace the original list with the sorted version, but I couldn't figure out which method was best for removing the entries and replacing them with a different ordered list. I ended up using a while loop, the hasChildNodes() method, and removeChild() methods to remove the old list and used appendChild() and insertAdjacentElement() to add the new list.

This gave me a good understanding of different math calculations within programming for real-life applications. I also got a good grasp on how to do other things such as animate images and customize drop-down lists.

Form and Quiz

This app let's users take a simple geography quiz and optionally submit some information along with it.

Key Features
  • Right sidebar for ad space or other extra content
  • Hints for giving correct input
  • Input validation on blur
  • Form input displayed to page after submission
  • Image given at end depends on number of correct answers
Skills Highlighted
  • Input validation
  • Displaying input to page
  • Conditional logic to display picture

In the making of this project, I was struggling with adding the input hints in the proper places. I also had trouble displaying the correct symbol to show whether the input was valid or needed to be fixed. In order to make it work, I placed a span tag to the right of each input box and, with JavaScript, used a switch statement to decide which hint text node it should insert into which span. The hint is offered when the input box is focused and the input is validated when the box is blurred.

Developing this app gave me a better grasp on making online quizzes, which involve conditional logic and incrementing values to total a score. It also helped me figure out how to give hints for filling in different types of input without visually overwhelming or confusing the user.
